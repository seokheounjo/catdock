1. 사전 준비

CatDock을 실행하려면 아래 3가지가 필요합니다.

필수 항목최소 버전확인 방법
Node.js18 이상node --version
pnpm9.15 이상pnpm --version
Claude Code CLI최신claude --version
1

Node.js 설치

아직 없다면 https://nodejs.org에서 LTS 버전을 다운로드하여 설치합니다.

터미널
$ node --version
v22.16.0
✓ Node.js 설치 확인 완료
2

pnpm 설치

Node.js 설치 후 터미널에서 아래 명령어를 실행합니다.

터미널
$ npm install -g pnpm
added 1 package in 3s
$ pnpm --version
9.15.4
3

Claude Code CLI 설치 & 로그인

Claude CLI를 설치하고 Anthropic 계정으로 로그인합니다.

터미널
$ npm install -g @anthropic-ai/claude-code
added 1 package in 8s
$ claude
→ 브라우저가 열리면 Anthropic 계정으로 로그인하세요
✓ Successfully authenticated!
Claude CLI 로그인이 가장 중요합니다. 로그인하지 않으면 에이전트들이 AI 응답을 받을 수 없습니다. claude 명령어를 실행하면 브라우저에서 로그인 화면이 열립니다.

2. 소스 받기 & 설치

1

GitHub에서 소스 코드 받기

터미널에서 원하는 폴더로 이동한 후 아래 명령어를 실행합니다.

터미널
$ git clone https://github.com/seokheounjo/catdock.git
Cloning into 'catdock'...
done.
$ cd catdock
2

의존성 설치

pnpm install로 필요한 패키지를 모두 설치합니다. (1~2분 소요)

터미널
$ pnpm install
Packages: +342
++++++++++++++++++++++++++++++++++++
dependencies installed in 45s
3

실행하기

pnpm dev로 앱을 실행합니다. 잠시 후 CatDock 창이 열립니다.

터미널
$ pnpm dev
vite v7.3.1 building...
✓ electron main process built successfully
✓ electron preload scripts built successfully
dev server running at: http://localhost:5173/
starting electron app...
빌드된 실행 파일로 사용하려면 pnpm build:win (Windows) / pnpm build:mac (macOS)을 실행하면 dist/ 폴더에 설치 파일이 생성됩니다.

1

CLI 확인

Claude Code CLI가 설치되어 있는지 자동으로 확인합니다. 녹색 체크가 뜨면 "다음"을 클릭합니다.

그룹 채팅 만들기

보고 최적화가 자동 적용됩니다. 짧은 응답, QUESTION 답변, 60초 내 중복 보고는 자동 생략되어 API 비용을 절약합니다.

10. 승인 게이트 활성화하기

1

대시보드 > 설정 탭 열기

활성화하면 에이전트가 작업 위임이나 에이전트 생성 전에 사용자 승인을 요청합니다.

13. 문제 해결 FAQ

Q: 에이전트가 갑자기 작업을 안 합니다.

A: 월 예산 초과일 가능성이 높습니다. 채팅에 "🚫 예산 초과" 메시지가 있는지 확인하세요. 에이전트 편집에서 예산을 늘리면 즉시 해제됩니다.

Q: 위임이 진행되지 않습니다.

A: 승인 게이트가 활성화되어 있고 응답을 안 했을 수 있습니다. 채팅 화면에 승인 요청 다이얼로그(파란색)가 떠 있는지 확인하세요. 120초 타임아웃 후 자동 거부됩니다.

Q: 비용이 너무 빠르게 증가합니다.

A: 보고 최적화는 자동 적용되지만, 여러 에이전트가 동시 위임을 받으면 비용이 빠릅니다. 예산을 설정하고, 승인 게이트로 위임을 통제하세요.

Q: 이번 달 비용은 어디서 확인하나요?

A: 대시보드 "팀 현황" 탭에서 각 에이전트 카드에 $0.0000 형태로 총 비용이 표시됩니다. 활동 피드에서 budget-warning/exceeded 이벤트도 확인하세요.

Q: 예산 설정 없이 쓰면 어떻게 되나요?

A: 예산 미설정 시 무제한으로 동작합니다. 기존과 동일하게 아무 제한 없이 작업합니다. 기존 사용자에게 영향 없습니다.

Q: 승인 게이트를 끄면 어떻게 되나요?

A: 기본값이 꺼짐(false)이므로, 기존과 동일하게 에이전트가 즉시 위임/생성합니다. 예산으로만 통제됩니다.

# 다크모드 기능 테스트 시나리오

## 📋 개요
Virtual Company 앱의 다크모드 기능 구현에 대한 QA 테스트 시나리오입니다.

**테스트 환경:**
- Electron 39 + React 19 + Tailwind CSS 4
- 여러 창 타입: Dock, Chat, Editor, Dashboard, GroupChat, Setup
- 설정 저장: config.json (`%APPDATA%/virtual-company-data/`)

## 🎯 핵심 테스트 시나리오

### 1. 테마 전환 기본 기능

#### 1.1 토글 버튼 동작
- **시나리오**: 다크모드 토글 버튼 클릭 시 즉시 테마 전환
- **예상 결과**: UI 전체가 라이트/다크 테마로 부드럽게 전환
- **테스트 포인트**: 깜빡임, 지연, 부분 적용 확인

#### 1.2 시스템 테마 자동 감지
- **시나리오**: OS 테마 설정 변경 시 앱 자동 반영
- **예상 결과**: 시스템 다크모드 ON/OFF에 따라 앱도 자동 전환
- **테스트 포인트**: 감지 지연, 무한 루프, 설정 충돌

#### 1.3 초기 테마 로딩
- **시나리오**: 앱 시작 시 저장된 테마 설정 적용
- **예상 결과**: 이전 세션의 테마 설정이 즉시 적용됨
- **테스트 포인트**: 기본값 처리, 로딩 순서, FOUC 현상

### 2. 여러 창 동기화

#### 2.1 창 간 실시간 동기화
- **시나리오**:
  1. Dock 창에서 다크모드 토글
  2. Chat 창, Editor 창이 열린 상태
- **예상 결과**: 모든 열린 창이 동시에 테마 전환
- **테스트 포인트**: IPC 메시지 전달, 동기화 지연, 일부 창 누락

#### 2.2 새 창 생성 시 테마 적용
- **시나리오**:
  1. 다크모드 상태에서 새 Chat 창 열기
  2. 라이트모드 상태에서 Editor 창 열기
- **예상 결과**: 새 창이 현재 테마로 즉시 렌더링
- **테스트 포인트**: 초기 렌더링 색상, 테마 불일치

#### 2.3 창 복원 시 테마 유지
- **시나리오**:
  1. 다크모드 설정
  2. 모든 창 닫기
  3. Dock에서 에이전트 채팅 다시 열기
- **예상 결과**: 복원된 창이 올바른 테마로 표시
- **테스트 포인트**: 상태 복원, 설정 불일치

### 3. 설정 저장 및 복원

#### 3.1 config.json 저장
- **시나리오**: 테마 변경 후 앱 종료 및 재시작
- **예상 결과**: 이전 테마 설정이 유지됨
- **테스트 포인트**: 파일 권한, 저장 실패, 기본값 처리

#### 3.2 설정 파일 손상 처리
- **시나리오**: config.json 파일 삭제 또는 손상
- **예상 결과**: 기본 테마(시스템 또는 다크)로 안전하게 복원
- **테스트 포인트**: 에러 핸들링, 기본값 적용, 복구 메커니즘

#### 3.3 설정 동시 수정
- **시나리오**: 여러 창에서 동시에 테마 변경 시도
- **예상 결과**: 마지막 변경사항이 적용되고 모든 창 동기화
- **테스트 포인트**: 경합 상황, 데이터 무결성

### 4. CSS 변수 전환

#### 4.1 커스텀 색상 전환
- **시나리오**: `--color-chat-bg`, `--color-accent` 등 CSS 변수 동적 변경
- **예상 결과**: 모든 커스텀 색상이 올바른 테마 값으로 전환
- **테스트 포인트**: 변수 업데이트 지연, 색상 충돌

#### 4.2 하드코딩 색상 대응
- **시나리오**: `bg-blue-900/60`, `text-white/80` 등 하드코딩 클래스 처리
- **예상 결과**: 테마에 맞는 적절한 색상으로 표시
- **테스트 포인트**: 대비 부족, 가독성 저하

### 5. 성능 및 UX

#### 5.1 전환 애니메이션
- **시나리오**: 테마 전환 시 부드러운 전환 효과
- **예상 결과**: 0.2-0.3초 내 부드러운 색상 전환
- **테스트 포인트**: 깜빡임, 화면 떨림, 전환 속도

#### 5.2 메모리 사용량
- **시나리오**: 테마 전환 반복 시 메모리 누수 확인
- **예상 결과**: 메모리 사용량 안정적 유지
- **테스트 포인트**: 메모리 누수, CSS 변수 정리

## ⚠️ 주요 엣지케이스

### 시스템 레벨 이슈
- **시스템 테마 변경 중 앱 시작**: 테마 감지 실패 가능성
- **접근성 고대비 모드**: OS 고대비 설정과의 충돌
- **다중 모니터**: 모니터별 다른 색상 프로필

### 앱 상태 이슈
- **에이전트 세션 중 테마 변경**: 채팅 기록 색상 불일치
- **Permission Dialog 표시 중**: 다이얼로그 색상 업데이트 누락
- **스트리밍 메시지 수신 중**: 실시간 메시지 색상 문제

### 데이터 무결성
- **config.json 동시 접근**: 여러 프로세스의 설정 파일 경합
- **IPC 메시지 손실**: 창 간 테마 동기화 실패
- **설정 롤백 실패**: 잘못된 설정 적용 후 복구 불가

## 🔧 테스트 도구 및 방법

### 수동 테스트
- 각 창 타입별 테마 적용 확인
- OS 테마 변경으로 자동 감지 테스트
- 설정 파일 직접 수정 테스트

### 자동화 테스트 후보
- Playwright E2E: 테마 전환 플로우 자동화
- Jest Unit: CSS 변수 변경 로직 테스트
- Electron Main Process: IPC 메시지 전달 테스트
